# design-consult-bot (Spring Boot)

Коротко:
- Бот ведёт пошаговый опрос: тип продукта → цель → референсы → бюджет/сроки → контакт.
- Итог пересылается в OWNER_CHAT_ID (или выводится в чат для теста).

Переменные окружения:
- TELEGRAM_BOT_TOKEN — токен от BotFather (обязательно)
- OWNER_CHAT_ID — chatId, куда пересылать лиды (опционально, если не задан — бот покажет результат в чате)

Запуск:
1. Убедитесь, что в pom.xml есть зависимость telegrambots-spring-boot-starter.
2. В терминале/Run configuration задайте env vars (TELEGRAM_BOT_TOKEN, OWNER_CHAT_ID).
3. Запустите:
   mvn spring-boot:run

Полезные команды в чате:
- /start — краткая справка
- /consult — начать консультацию
- /getid — получить ваш chatId (полезно чтобы узнать OWNER_CHAT_ID)

Дальше можно:
- добавить inline-кнопки вместо свободного текста,
- сохранять лиды в БД (JPA/Postgres),
- принимать изображения (референсы) и сохранять в S3/GCS,
- переключиться на webhook для продакшна.
